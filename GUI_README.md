# 图形界面使用说明 (GUI Usage Guide)

## 功能特性

本图形界面提供了以下功能：

- **直观的站点选择**: 下拉菜单显示所有可用站点，支持自动排序
- **实时路线查询**: 快速计算两点间的最短路线
- **详细的结果展示**: 显示完整路线、耗时和站点数
- **后台处理**: 查询过程在后台线程运行，不会冻结UI
- **多语言支持**: 英文和中文双语界面

## 使用步骤

### 1. 前置条件
确保已经运行过 `Dataprocess.py` 生成了 `metro_graph.pkl` 文件：
```bash
python3 Dataprocess.py
```

### 2. 启动GUI
```bash
python3 GUI.py
```

### 3. 查询路线
1. 在"起点"下拉菜单中选择出发站点
2. 在"终点"下拉菜单中选择目的站点
3. 点击"查询最短路线"按钮
4. 等待结果显示（一般在几秒内）

### 4. 查看结果
- **总耗时**: 以秒和分钟显示
- **站点数**: 完整路线经过的站点总数
- **完整路线**: 列表形式显示每一站的ID和名称

## 界面说明

| 组件 | 说明 |
|------|------|
| 起点/终点选择框 | 使用下拉菜单选择地铁站 |
| "查询最短路线"按钮 | 触发Dijkstra算法计算最短路径 |
| "清空"按钮 | 重置所有选项和结果 |
| 结果显示框 | 显示完整的路线和统计信息 |

## 技术实现

- **GUI框架**: Python tkinter (内置库)
- **多线程**: 使用threading避免UI冻结
- **数据加载**: 通过pickle反序列化预处理的图数据
- **算法**: 调用Dijkstra.py中的dijkstra函数

## 常见问题

**Q: 界面启动后没有显示任何站点？**
A: 确保 `metro_graph.pkl` 文件存在，并且是由最新的 `Dataprocess.py` 生成的。

**Q: 查询时出现"无法到达"的提示？**
A: 这表示选定的两个站点之间没有连接路径，或者输入的站点ID不正确。请确认站点ID的有效性。

**Q: 如何显示站点的中文名称？**
A: 运行 `Dataprocess.py` 时会自动生成 `station_names.pkl` 文件，GUI会自动加载并显示站点名称。
